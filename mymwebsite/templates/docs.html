{% trans "Into Details" %}

{% trans "1. Verification Points - VPs" %}

{% trans "A VP (Verification Point) is each of the visual targets for the capture of images (pictures). Each VP may have one or several TAGs." %} {% trans "A VP may be a photo of pretty much anything. You will have to assess what can be a standard VP or if it is needed to use one our possible alternative options to capture the required image." %} {% trans "For instance, a VP may be an image of an equipment panel, digital gauges, analog gauges, computer and other devices screens, compliance items etc, and even portable VPs to follow on going jobs and specific problems." %}

{% trans "Standard VPs" %}

{% trans "This is the most common type, where the VP image is enough to identify the subject perfectly." %}

{% trans "Ambiguous VPs" %}

{% trans "How to differentiate panels that are almost the same? How to tell apart one main equipment and its backup?" %} {% trans "To solve this ambiguity problem we developed a specific technology." %} {% trans "In this case you will need to apply small markers (normally to be printed in adhesive) to the subjects to be captured in order to be able to tell them apart." %} {% trans "The markers are small 45mm squares that the smartphone will use to identify images that otherwise would be impossible to differentiate." %}

{% trans "Super VPs" %}

{% trans "One obvious downside of the image recognition technology is that you need an image to recognize." %} {% trans "That means that the VP subject cannot change completely, as the smartphone would be unable to recognize an image that no longer exists." %} {% trans "To solve this problem we developed another specific technology." %} {% trans "In this case we use an specific image as a subject of our VP. This subject is then called (very humbly) by us a super VP." %} {% trans "So, using this super VP you will be able to capture images that change a lot, that change color, or have large parts included or excluded." %} {% trans "The smartphone will always capture the image in the same relative position initially configured for each specific super VP." %}

{% trans "2. TAGs" %}

{% trans "A TAG is the real data contained within a VP." %} {% trans "The TAG will always be a numerical value, so you have to includ this concept when preparing your TAG questions." %} {% trans "For instance, in a photo of a certain equipment control panel (that would be a VP) we can have multiple TAGs: the hour-meter, the tachometer etc." %} {% trans "In an image of compliance items, we can check if an area is tidy as it should be (10 if perfect, 8 if OK, for example), or if all safety equipment are in the right place etc." %} {% trans "For each and every TAG you will define a tag question, that will be answered by you or your team when processing the VPs images." %}
